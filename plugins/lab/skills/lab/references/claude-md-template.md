# CLAUDE.md 生成テンプレート

`.lab/CLAUDE.md` を生成するためのテンプレート。
研究の「名刺」。ルールや手続きはSKILL.mdが持つ。
`{{...}}` の変数はオンボーディングデータで置換する。

---

## テンプレート

````markdown
# Lab — 研究室

## 研究概要

- **研究テーマ**: {{RESEARCH_TOPIC}}
- **現在のフェーズ**: {{CURRENT_PHASE}}
- **作成日**: {{CREATED_DATE}}

## チーム構成

```
教授（思考・判断・解釈・壁打ち・論文執筆・STATE.md/log.md管理）
 │
 ├── 理論家（証明構築・定式化・数学的議論）
 ├── 実験家（コード実装・実験実行・結果可視化）
 ├── 文献調査員（論文要約・サーベイ・引用ネットワーク・BibTeX管理）
 └── レビュアー（批判的評価 + 新しい洞察の発見）
```

## 基本ルール

- STATE.mdの更新は報告の前に行う
- STATE.mdは省略せず根拠・数値・考察を十分に記録する
- 壁打ち中の重要な洞察は教授STATE.mdに即座に書く
- ファイル命名：日次は `YYYY-MM-DD.md`、トピックは `kebab-case.md`
- 既存ファイルは上書きしない（追記 or 新規作成）

## MCP

Semantic Scholar API + Perplexity。未設定でも動くが文献調査の精度が落ちる。

## パーソナライズメモ

{{PERSONALIZATION_NOTES}}
````

## 変数リファレンス

| 変数 | ソース | 説明 |
|------|--------|------|
| `{{RESEARCH_TOPIC}}` | Q1 | 研究テーマ |
| `{{CURRENT_PHASE}}` | Q2a | 探索 / 仮説検証 / 論文執筆 |
| `{{CREATED_DATE}}` | 自動 | 作成日 |
| `{{PERSONALIZATION_NOTES}}` | Q1+Q2+Q3 | ヒアリング全体から生成したメモ |
